# Author Homepage Skill

为一个研究者生成类似 research homepage 的 Obsidian 笔记，重点整理：

- 基本身份信息
- 教育经历
- 研究方向
- 代表作
- 按领域分类的论文
- 按时间线分类的学术轨迹
- 人物关系（导师、学生、合作者、机构）
- 内部 wikilinks，便于在 vault 中形成作者知识网络

## 何时使用

当用户：

- 只给你一个作者名字，希望你“搜他的文章并整理主页”
- 想知道某位学者的代表作、研究方向、时间线、学术关系
- 想把作者信息沉淀成 vault 里的长期知识节点
- 提到“作者主页”“research homepage”“作者卡片”“学术人物页”“导师关系”“合作网络”

就使用本 skill。

## 默认输出位置

- 主目录：`20_Research/People/`
- 默认文件名：`20_Research/People/[作者名].md`

不要把这类笔记放进：

- `20_Research/Papers/`（这是论文笔记目录）
- `03_阅读/`（这是阅读过程目录）

## 推荐信息来源

优先从以下来源交叉确认：

1. 作者个人主页 / 实验室主页
2. Google Scholar
3. DBLP
4. Semantic Scholar
5. OpenReview / arXiv / 机构主页
6. 论文第一页作者单位与致谢

如果不同来源冲突：

- 优先信任作者个人主页和当前机构主页
- 其次信任 Google Scholar / DBLP
- 在笔记中注明“信息可能已变化”

## 工作流

### 1. 识别作者

先确定：

- 中文名 / 英文名
- 是否存在同名学者
- 所属领域是否与你的 vault 方向相关

如遇同名，优先根据：

- 研究方向
- 机构
- 代表论文

做消歧，并在笔记中注明你选的是哪一位。

### 2. 收集作者基本信息

至少收集以下字段：

- 姓名
- 当前职称 / 身份
- 当前机构
- 实验室 / 中心 / 院系
- 教育经历（本科 / 硕士 / 博士 / 博后，若可获得）
- 主要研究方向
- 主页链接
- Google Scholar / DBLP / Semantic Scholar 链接（若可获得）

### 2.1 教育经历整理要求

教育经历是必选部分，不要省略。

至少尽量整理：

- 本科院校与专业
- 硕士院校与专业
- 博士院校与项目
- 博后机构 / 合作导师

如果某一段缺失：

- 写明“暂未查到”
- 不要用模糊猜测替代

### 3. 代表作筛选原则

不要简单罗列所有论文。优先选择：

- 被引高
- 对领域影响大
- 能代表不同研究阶段
- 能代表不同研究方向分支
- 与用户关注方向最相关

建议默认给出：

- `5-10` 篇代表作

并为每篇给出：

- 标题
- 年份
- venue
- 一句话贡献
- 是否已在本 vault 有笔记（若有，用 wikilink）

### 4. 按领域分类

把作者工作整理成 `2-5` 个主题桶，例如：

- 神经动力学
- 低秩 RNN
- 脑机接口
- 认知神经科学
- 表示学习

每个桶下列：

- 代表论文
- 该方向的核心问题
- 作者在该方向的独特贡献

### 5. 按时间线分类

把作者轨迹分成若干时期，例如：

- 早期工作
- 中期代表作
- 最近方向

每个时期总结：

- 主要问题
- 方法风格
- 代表论文
- 与前一阶段相比发生了什么变化

不要只按年份堆列表，要写出“研究主线的演化”。

### 6. 人物关系与 Obsidian 连接

尽量建立以下 wikilinks：

- 导师 / 学术祖先
- 学生 / 共同作者
- 所在实验室
- 所在机构
- 你 vault 里已经存在的相关作者或概念

如果某个关系不确定：

- 写“可能合作频繁 / 可能师承关系，需进一步核实”
- 不要伪造关系

### 7. 与本 vault 的连接方式

应主动链接到：

- 已有作者笔记
- 已有方法概念笔记
- 已有论文笔记

例如：

- `[[低秩RNN]]`
- `[[神经动力学]]`
- `[[某篇论文标题]]`
- `[[某位合作者]]`

## 输出格式要求

始终输出 Obsidian Markdown。

### Frontmatter

使用类似结构：

```yaml
---
title: 作者名
aliases:
  - 英文名或常见写法
tags:
  - 作者
  - 研究者
  - 学术主页
  - <主领域>
created: YYYY-MM-DD
status: evergreen
---
```

### 默认笔记结构

```markdown
# 作者名

## 一句话简介

## 基本信息

## 教育经历

## 研究关键词

## 研究主线概览

## 代表作

## 按领域整理

## 按时间线整理

## 人物关系

## 相关机构与实验室

## 与本库的连接

## 外部链接

## 备注
```

## 写作要求

- 不要只做“百科式简历”
- 要更像“研究主页 + 学术地图”
- 重点是帮助用户快速理解：
  - 这个人做什么
  - 最值得看什么
  - 和哪些方向/人相关
  - 研究主线怎么演化
- 教育经历不仅要列学校，还要尽量体现训练路径如何影响研究方向演化

### 代表作部分

建议格式：

```markdown
1. **论文标题** (Venue, Year)
   - 贡献：一句话总结
   - 关键词：...
   - 关联：[[相关概念]] / [[相关论文]]
```

### 人物关系部分

建议拆成：

- 导师 / 学术来源
- 常见合作者
- 学生 / 同门（若可确认）
- 所属机构网络

### 与本库的连接

至少列出：

- 已有相关论文笔记
- 已有相关概念笔记
- 已有相关作者笔记

## 质量要求

完成后自查：

- 是否有基本身份信息
- 是否有教育经历
- 是否给出了代表作，而不是仅列 paper 列表
- 是否按领域分类
- 是否按时间线梳理
- 是否建立了人物关系和机构关系
- 是否使用了 Obsidian wikilinks
- 是否保存到 `20_Research/People/`

## 边界情况处理

- **同名学者** → 在「识别作者」阶段先枚举候选（方向 + 机构 + 代表作），让用户确认；未确认前不要写入笔记。
- **个人主页缺失** → 回退到 Google Scholar / DBLP / Semantic Scholar / OpenAlex 交叉核对；在笔记中标注「无主页，按公开学术索引整理」。
- **Scholar 访问受限 / 429** → 切到 DBLP / OpenAlex / Semantic Scholar 补全；不要循环重试同一来源。
- **教育经历查不到某段** → 写「暂未查到」，不要用常见路径猜测替代（例如默认博士→博后→助理教授）。
- **同名同机构的人物关系不确定** → 标注「需核实」，不要把合作者伪造成导师或学生。
- **已有笔记** → 不要覆盖；打开后在末尾追加新信息或与用户确认是否刷新。

## 适合的默认动作

如果用户只给名字，不额外说明：

1. 搜索该作者主页与论文列表
2. 提取代表作与研究方向
3. 生成一篇 research-homepage 风格的作者笔记
4. 保存到 `20_Research/People/[作者名].md`

如果用户给出额外方向，例如“重点看他在 SEEG 上的工作”：

- 在 `按领域整理` 中优先展开该方向
- 代表作优先选择该方向的工作
